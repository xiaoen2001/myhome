# 🎬 项目演示指南

## 答辩演示准备

### 启动服务（演示前 5 分钟）

```bash
# 终端1：确保 MySQL 运行中
# 终端2：启动后端
cd share
mvn spring-boot:run

# 终端3：启动前端
cd read/read
npm run dev
```

## 演示脚本（6~8 分钟）

### 第一幕：项目概览（1 分钟）

> "大家好，我做的毕设项目是**Read 社区——学生读书笔记分享平台**。
> 这是一个面向大学生的读书社交平台，用户可以写笔记、交书友、发现好书。
> 技术栈是 **Vue 3 + Spring Boot 3.5 + MySQL 8.4**，前后端分离架构。"

**操作**：打开首页 → 展示轮播图、推荐笔记

### 第二幕：用户注册与登录（30 秒）

> "首先演示用户注册和登录。系统使用 JWT 无状态认证，密码 BCrypt 加密存储。"

**操作**：
1. 点击右上角"注册"
2. 输入用户名、密码、邮箱
3. 注册成功自动跳转首页

### 第三幕：写笔记（1.5 分钟）

> "平台核心功能是笔记管理。我演示一下完整的笔记发布流程。"

**操作**：
1. 导航栏 → "我的笔记" → "新建笔记"
2. 填写标题、书名、选择分类
3. 使用富文本编辑器写内容（wangEditor）
4. 上传封面图片
5. 选择公开可见 → 提交审核
6. **切换 admin 账号**：
7. 后台管理 → 笔记管理 → 找到待审核笔记 → 点击"审核通过"
8. 笔记公开发布

### 第四幕：社交互动（1.5 分钟）

> "平台支持完整的社交功能：点赞、收藏、评论、好友聊天。"

**操作**：
1. 回到前台 → 笔记列表 → 点开一篇笔记
2. 展示**评论**功能（已有 100 条评论）
3. 点击**点赞**、**收藏**
4. 导航栏 → "我的好友" → 展示好友列表
5. 点击好友 → 打开**实时聊天**（WebSocket 推送）
6. 发送一条消息，展示即时到达

### 第五幕：通知系统（1 分钟）

> "通知系统支持全局公告和个人通知。审核结果会实时推送给作者。"

**操作**：
1. 点击铃铛图标 → 通知列表
2. 展示全局系统通知（社区公告）
3. 展示个人通知（审核通过、点赞提醒）
4. **切换到其他用户**：展示只有该用户的通知，看不到其他人的

### 第六幕：管理后台（1 分钟）

> "管理员拥有完整的后台管理功能，包括数据仪表盘。"

**操作**（admin 账号）：
1. 导航 → 后台管理
2. **仪表盘**：展示用户数、笔记数、ECharts 趋势图
3. **笔记管理**：审核通过/驳回（展示驳回后作者收到通知）
4. **用户管理**：用户列表、启用/禁用
5. **轮播图管理**：上传本地图片、排序、状态切换
6. **分类/标签管理**：增删改查

### 收尾：技术亮点（30 秒）

> "总结一下项目的技术亮点：
> 1. 前后端分离，JWT 无状态认证
> 2. WebSocket 实时聊天和通知推送
> 3. 个人通知 vs 全局通知的分层设计
> 4. MyBatis-Plus Lambda 查询 + 分页
> 5. 数据库 14 张表，冗余计数 + 唯一索引保证数据一致性"

### 面试官会关注的点

1. **数据库设计** → 展示 14 张表的 ER 关系 + 索引策略
2. **安全设计** → JWT + BCrypt + 权限控制 + 敏感配置隔离
3. **代码规范** → 分层架构、接口/实现分离、Lombok 简化代码
4. **项目文档** → README + 架构文档 + API 文档

### 常见追问及回答

**Q: 为什么选 Vue 3 而不是 React？**
A: Vue 3 的 Composition API 与 Spring Boot 的概念相似（组合优于继承），学习曲线平缓，Element Plus 生态成熟。

**Q: JWT Token 过期怎么处理？**
A: 前端 Axios 拦截器捕获 401，清除 Token 并跳转登录页，用户无感知。

**Q: 评论和点赞的高并发怎么处理？**
A: 使用数据库唯一索引防止重复操作，计数通过 Service 层事务更新而非触发器，保证一致性。

**Q: 文件和图片怎么存储的？**
A: 开发环境存储到本地磁盘，通过 Spring 静态资源映射访问。生产环境可对接 OSS（阿里云/七牛）。

## 备用：故障预案

| 问题 | 解决方案 |
|------|----------|
| MySQL 连不上 | 检查服务：`Get-Service MySQL`，确保状态 Running |
| 端口被占用 | `netstat -ano | findstr 8081`，换端口 |
| 前端白屏 | 检查 Vite 是否启动，按 F12 看控制台报错 |
| 登录失败 | 检查后端是否启动，确认数据库有用户数据 |
